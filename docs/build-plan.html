Parallelism rules distilled from the hackathon retrospective

  • Max 2 concurrent workers on this scope. Three or more = coordination loss > parallelism gain (we hit this at 5 in the hackathon).
  • One worker owns each shared file. If two must touch the same file, they go serial.
  • Every prompt includes: file pointers with section names · forbidden patterns list · explicit IPC channel names · "what other lanes will produce" contract · integration smoke test in DoD · STOP_IF clause.
  • Workers commit and push every meaningful step. No rebase. No co-signing.
  • Smoke test through the integration boundary — not "typecheck passes" but "ran the app, clicked the thing, saw the expected effect in console + UI."
