Parallelism rules from the hackathon retrospective

  • Max 3 concurrent workers at any time. Each owns a non-overlapping file set per docs/contracts.md § 8.
  • Contracts first, code second. If a shape isn't in contracts.md yet, propose a doc commit before writing code.
  • Verification through the integration boundary is non-negotiable per contracts.md § 10.
  • Review gates pause execution. Workers do not dispatch into the next phase until the orchestrator confirms the prior gate passed.
  • No rebase, no co-signing. git add → commit → push.
